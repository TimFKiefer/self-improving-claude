## Task 1: Cost estimation helper

**Why:** Per-iteration USD lookahead lets the loop abort *before* exceeding `--max-usd` rather than after.

- [ ] **Step 1.1:** In `evals/auto_loop.py`, add coarse per-model token prices (input/output, USD per 1M tokens). Source from Anthropic public pricing as of 2026-05; document in code comment that prices may drift.

```python
# Anthropic API pricing (USD per 1M tokens), input / output, as of 2026-05.
# Coarse approximation — claude --print exit metadata doesn't reliably expose USD,
# so we estimate from char counts (chars/4 ≈ tokens). Re-calibrate if pricing changes.
MODEL_PRICING = {
    "haiku":              {"in":  1.00, "out":  5.00},
    "claude-sonnet-4-5":  {"in":  3.00, "out": 15.00},
    "opus":               {"in": 15.00, "out": 75.00},
}
DEFAULT_PRICING = {"in": 3.00, "out": 15.00}  # fallback for unknown model

def _estimate_eval_cost_usd(skill_model: str, judge_model: str, n_fixtures: int = 1) -> float:
    """Coarse per-eval USD estimate. One fixture invocation ≈ 50k input + 5k output tokens
    for the skill-runner, plus ≈ 2k input + 1k output for the judge per proposal."""
    skill = MODEL_PRICING.get(skill_model, DEFAULT_PRICING)
    judge = MODEL_PRICING.get(judge_model, DEFAULT_PRICING)
    skill_cost = (50_000 * skill["in"] + 5_000 * skill["out"]) / 1_000_000
    judge_cost = (2_000 * judge["in"] + 1_000 * judge["out"]) / 1_000_000
    return n_fixtures * (skill_cost + judge_cost)

def _estimate_proposer_cost_usd(proposer_model: str) -> float:
    """Coarse per-proposer-call USD estimate. ≈ 10k input + 500 output."""
    p = MODEL_PRICING.get(proposer_model, DEFAULT_PRICING)
    return (10_000 * p["in"] + 500 * p["out"]) / 1_000_000
```

- [ ] **Step 1.2:** Per-iteration USD lookahead:

```python
def _estimate_iteration_cost_usd(*, skill_model: str, judge_model: str,
                                  proposer_model: str, holdout_gate_on: bool) -> float:
    """Upper-bound estimate for one iteration: proposer + visible single-fixture eval
    + (if gate on) held-out 3-fixture eval. Use the upper-bound to be conservative."""
    cost = _estimate_proposer_cost_usd(proposer_model)
    cost += _estimate_eval_cost_usd(skill_model, judge_model, n_fixtures=1)  # visible target
    if holdout_gate_on:
        cost += _estimate_eval_cost_usd(skill_model, judge_model, n_fixtures=3)  # held-out
    return cost
```

- [ ] **Step 1.3:** Tests in `test_auto_loop.py`:
  - Estimates are positive
  - Sonnet costs more than haiku (sanity)
  - Opus costs more than sonnet
  - Iteration estimate ≈ proposer + visible + holdout (when gate on)
  - Iteration estimate skips holdout when gate off

- [ ] **Step 2.1:** Add CLI flag in `main()`:

```python
                    help="abort cleanly before next iteration would exceed this USD cap")
```

- [ ] **Step 2.2:** Track cumulative spend in `state`:

```python

```

- [ ] **Step 2.3:** Per-iter pre-check (before pick_target / proposer / eval):

```python
if args.max_usd is not None:
    lookahead = _estimate_iteration_cost_usd(...)
    if state["usd_spent"] + lookahead > args.max_usd:
        print(f"[cap] would exceed --max-usd {args.max_usd:.2f} "
              f"(spent={state['usd_spent']:.2f} next iter ~={lookahead:.2f}); aborting.",
              file=sys.stderr)
        break
```

- [ ] **Step 2.4:** Per-iter accumulation:


## Risk-specific mitigations for v0.5.0

| Risk | Mitigation |
|---|---|
| 50-iter run hits API rate limits | β's 20-iter completed without hitting limits; 50 should too. If hit, the cap-abort path means clean partial result; resume = v0.7. |
| Cost overrun | Hard `--max-usd 50` cap. Conservative lookahead estimate ($0.40-0.60/iter) before each iter. |
| Loop discovers and ships a regression on a held-out-pass case | Held-out gate is the load-bearing safety. β proved it works (9 catches in 20). 50-iter expected to catch ~22. |
| Pricing in `MODEL_PRICING` drifts vs Anthropic actual | Coarse estimate — overshoot is OK (errs toward early stop). Comment in code flags re-calibration. |
| Cost estimate doesn't match actual subscription billing | We can't see subscription billing from inside the CLI. Estimate is the best we have; document the limitation in CHANGELOG. |
