# v0.4.0 — Knowledge Re-grounding + Single-Source Skills (Track 7) — Design Spec

- **Date:** 2026-05-27
- **Status:** Approved in brainstorm; awaiting user review before `writing-plans`.
- **Milestone:** v0.4.0 "Foundation for Autonomy", **Track 7** — single source of truth.
- **Scope:** a generator that single-sources the duplicated orchestrator procedure **and** the 5 reference docs from canonical `plugin/skills/_shared/` sources; a pre-commit guard that makes drift impossible; and a rewrite/expansion of `docs/knowledge/`. The runtime content of the skills does **not** change (byte-identical refactor).
- **Companion:** `docs/superpowers/plans/2026-05-24-self-improving-claude-v0.4-backlog.md` §7.

---

## 1. Why this work exists

Two problems, both flagged by the v0.3.2 analysis and confirmed this session:

- **Drift risk (duplication).** The Steps 1–10 orchestrator body is duplicated **verbatim** in both `plugin/skills/improve/SKILL.md` and `plugin/skills/improve-init/SKILL.md`, and the **5 reference files are byte-identical copies** in `improve/references/` and `improve-init/references/`. Every v0.3.x fix had to be mirrored by hand. The v0.3.0 CHANGELOG records the cause: *"The probe for `@`-mention resolution outside the skill directory FAILED, so the implementation took the inline duplication fallback … Future changes … must be applied to BOTH copies."* The ROADMAP makes v0.4 Track 7 the point where *"drift is structurally impossible."*
- **Stale knowledge.** `docs/knowledge/plugins-and-skills.md` still shows the **abandoned** `skills/self-improving-claude/` single-skill layout, lists as **open** questions that v0.3 answered (marketplace install works; the manifest is `plugin/.claude-plugin/plugin.json`; hooks register via the plugin, not auto), and **never records** the `@../` non-resolution. Seven facts the plugin *relies on* are uncaptured.

---

## 2. Goals

- **G1.** Exactly **one** canonical copy of (a) the orchestrator procedure and (b) each of the 5 references. Both `SKILL.md` files and their `references/` dirs are **generated** from these.
- **G2.** **Byte-identical first run.** The generator is seeded by *extracting* canonical sources from the current files; regenerating reproduces the current committed `SKILL.md` + `references/` **exactly** (zero `git diff`). Runtime behavior, the sandbox/legacy eval, and the baselines are therefore unaffected — no re-baseline.
- **G3.** **Drift impossible.** A pre-commit check (`scripts/sync_skills.py --check`) fails the commit if any generated file was hand-edited.
- **G4.** `docs/knowledge/plugins-and-skills.md` rewritten to current reality; the 7 relied-upon facts captured (in plugins-and-skills.md + hooks-and-sdk.md).
- **G5.** Generator fully unit-tested.

## 3. Non-goals

- **No wording/behavior change** to the skills — this is a pure refactor (G2). Improving the procedure text is separate, later work.
- **`improve-uninstall`** — a separate skill with no shared procedure/references; untouched.
- **Generating references *from* `docs/knowledge/`** (floated in backlog §7.2). Deliberately **not** done: knowledge docs are full design rationale; references are trimmed runtime instructions — transforming one into the other is a lossy mini-project. Single-sourcing within `plugin/skills/_shared/` achieves the no-drift goal without that transform. (Divergence noted.)
- **`@../` adoption** is conditional on the Task-0 re-probe; the default is copy-via-generator.

## 4. Architecture

### 4.1 Canonical sources (one copy each)

```
plugin/skills/_shared/
  orchestrator-procedure.md         # "# self-improving-claude — Orchestrator": How-to-operate,
                                     #  Grounding (@references/...), Steps 1-10 — extracted verbatim
  references/
    prompt-rubric.md  hook-patterns.md  tools-reference.md  settings-merge.md  examples.md
  preambles/
    improve.md                       # frontmatter + intro + "## Inputs" (reactive) — per-skill
    improve-init.md                  # frontmatter + intro + "## Inputs" (proactive) — per-skill
```

The per-skill **preamble** (frontmatter, intro line, the mode-specific `## Inputs` block, and the `## Procedure to follow` header) legitimately differs between the two skills and is kept as a canonical per-skill file. Everything below the procedure header is the shared body.

### 4.2 Generator `scripts/sync_skills.py`

- **`build` (default):** for each skill in `{improve, improve-init}`: write `SKILL.md = preambles/<skill>.md + "\n" + orchestrator-procedure.md`; and materialize `plugin/skills/<skill>/references/*.md` from `_shared/references/*.md`. Idempotent.
- **`--check`:** regenerate into memory and compare to the on-disk files; exit non-zero (printing the offending paths) if any differ. No writes.
- **Seeding (the safety invariant):** the canonical sources are produced by splitting the *current* `SKILL.md` files at the stable boundary (the `## Procedure to follow` / `# self-improving-claude — Orchestrator` line) — head→`preambles/<skill>.md`, tail→`orchestrator-procedure.md` (the tail is asserted identical across both skills before being accepted as shared), and copying the current references into `_shared/references/`. The implementation plan's **first task asserts `git diff` is empty after a build** — if it isn't, the split is wrong and we stop.
- **`@../` Task 0 (gating spike):** probe whether `@../_shared/references/foo.md` resolves from a skill in the installed Claude Code. If **yes**, the generated body can `@`-include the shared dir and the generator need not copy reference files (true single physical copy). If **no** (the v0.3 finding), the generator copies `_shared/references/*` into each skill's `references/` so the existing `@references/foo.md` mentions keep resolving. Either way `_shared/` is the single source; the probe only decides copy-vs-link. Default to **copy** unless the probe clearly passes.

### 4.3 Drift guard

- A repo `pre-commit` hook (`.git/hooks/pre-commit`, installed by a documented one-liner or a `scripts/install-hooks.sh`) runs `python3 scripts/sync_skills.py --check` and blocks the commit on mismatch. Documented in `README.md`/CONTRIBUTING.
- A note for a future CI step running the same `--check` (when the repo goes public). Not built now.

### 4.4 Knowledge re-grounding

- **Rewrite `docs/knowledge/plugins-and-skills.md`** to current reality: the real `plugin/` layout; marketplace install + `plugin/.claude-plugin/plugin.json` manifest + root `marketplace.json`; hooks register via the plugin (installed by `/improve-init`), not auto; intra-directory `@references/...` resolves while **`@../` does not**. Remove the abandoned `skills/self-improving-claude/` layout and the settled "Pattern A/B" debate.
- **Capture the 7 relied-upon facts** (in plugins-and-skills.md and/or hooks-and-sdk.md as topically fits): (1) `type:"prompt"` hook contract (`permissionDecision`/JSON output); (2) advanced JSON hook stdout output; (3) `Stop` force-continue + `stop_hook_active`; (4) `${CLAUDE_PLUGIN_ROOT}` / `${CLAUDE_PROJECT_DIR}`; (5) `marketplace.json` schema; (6) `hooks.json` plugin-hook registration; (7) `@../` non-resolution (with the v0.3.0 CHANGELOG citation). Sourced from the skill `references/` (which already rely on them) + verified behavior.

## 5. Testing

- `scripts/tests/test_sync_skills.py` (or `evals/tests/…` per repo convention): `--check` passes on the clean committed tree; after simulating a hand-edit in a `tmp` copy of a generated file, `--check` fails and names it; `build` is idempotent (running twice = no change).
- **The decisive manual gate (plan Task 1):** after extracting canonical sources and running `build`, `git diff` over `plugin/skills/{improve,improve-init}/**` is **empty** — proving the refactor is behavior-preserving. Only then commit.

---

## 6. Risks & mitigations

| Risk | Mitigation |
|---|---|
| Extraction split is subtly wrong → generated `SKILL.md` differs | The byte-identical-`git diff`-empty assertion (G2 / Task 1) catches it before any commit; stop and fix the split |
| The eval drives the generated `SKILL.md` | It's byte-identical to today's, so the sandbox/legacy eval + the 2026-05-26 baselines are unaffected (no re-baseline) |
| `@../` re-probe inconclusive | Default to copy-via-generator (safe; what v0.3 already relies on) |
| Pre-commit hook is local-only (not enforced for other clones until CI) | Acceptable for a solo/local repo; CI `--check` noted as a follow-up |
| Knowledge rewrite introduces a new inaccuracy | Cross-check each captured fact against the skill `references/` that already use it + the CHANGELOG/specs |

---

## 7. Out of scope / deferred

- Any change to skill *wording*/behavior; `improve-uninstall`; generating references from `docs/knowledge/`; a CI `--check` job (local pre-commit only for now).
